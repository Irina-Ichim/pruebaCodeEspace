Cómo Empezar

Para crear un Codespace, ve a tu repositorio de GitHub, haz clic en el botón Code y selecciona Create Codespace y la rama. Espera a que se configure el entorno y estarás listo para comenzar a programar.

Características Clave

  • Entorno de desarrollo completamente personalizable.
  • Extensiones y herramientas preinstaladas.
  • Integración directa con repositorios de GitHub.
  • Capacidad de compartir puertos para acceder a servidores locales.

Consejos

Usa la terminal incorporada para comandos rápidos, instala extensiones para mejorar la productividad y utiliza la pestaña Ports para acceder a servidores locales.

Colaboración en Codespaces

Para colaborar, utiliza la extensión Live Share. Invita a otros compartiendo el enlace generado, lo que permite la edición y depuración en tiempo real.

Una vez que termines de programar, puedes guardar tus cambios usando los siguientes comandos:

  • git add . (agrega todos los archivos al área de preparación).
  • git commit -m "Descripción de los cambios" (guarda los cambios en un commit).
  • git push (envía los cambios al repositorio remoto).

Si deseas previsualizar tu proyecto en un navegador, puedes usar los siguientes comandos para iniciar un servidor local:

  • npm install -g http-server (instala un servidor HTTP simple).
  • http-server . (inicia el servidor en el directorio actual).

Utilizamos ramas

Para trabajar en Codespaces desde ramas y facilitar la colaboración, sigue este flujo de trabajo:

Flujo con Ramas y Pull Requests

  1. Crear una rama nueva: Usa el comando git checkout -b nombre-de-la-rama para crear una rama específica.
  2. Trabajar en la rama: Haz los cambios necesarios y guárdalos con git add y git commit.
  3. Subir cambios: Usa git push origin nombre-de-la-rama.
  4. Crear un Pull Request: Desde GitHub, crea un Pull Request para fusionar los cambios en main. Revisa y aprueba antes de fusionar.

Flujo Simplificado sin Pull Requests

  1. Crear una rama: Usa git checkout -b nombre-de-la-rama para trabajar de forma aislada.
  2. Colaborar directamente: Todos los colaboradores trabajan en la misma rama dentro del Codespace.
  3. Guardar cambios: Cada persona guarda sus progresos usando git add, git commit y git push origin nombre-de-la-rama.
  4. Fusionar la rama: El encargado del proyecto puede hacer un git merge nombre-de-la-rama desde main una vez que los cambios estén listos.

Este enfoque asegura que el trabajo colaborativo sea organizado y eficiente, adaptándose al nivel de experiencia del equipo.
